1. O RESUMO DE OURO (A Lógica do 'ret' e dos Getters)
-------------------------------------------------------------------
Você descobriu que a variável 'ret' não guarda apenas o nome e a 
idade soltos. Ela guarda o PACOTE COMPLETO: as propriedades 
(nome, idade) e também as funções (getnome, getidade).

Quando você usa o 'presultado.push(ret)', o array passa a guardar 
uma lista desses pacotes fechados na memória.

O seu HTML é "burro" e não consegue ler um pacote fechado direto 
(por isso ele mostra [object Object]). Então, dentro do .map(), 
nós usamos o 'el.getnome()' e 'el.getidade()' para abrir esse 
pacote específico, pegar o valor lá de dentro e entregar em texto 
puro para o HTML conseguir mostrar na tela. 

O console.log consegue abrir o pacote sozinho no F12, mas o HTML 
da página precisa que você abra para ele usando o 'return' dos gets!

adicionais importantes









A resposta correta é a sua segunda opção: é porque dentro de cada elemento do array existe a propriedade this.nome. A posição dele no array não tem nada a ver com o fato de ele saber quem é o nome ou a idade.

Para você entender perfeitamente, pense no seguinte:

O Array é uma Garrafeira (uma caixa de caixas)
Quando você faz presultado.push(ret), imagine que você está colocando uma caixa trancada dentro de uma gaveta do array.

Na gaveta 0 do array, está o ret do Lucas.

Se você adicionar outra pessoa, na gaveta 1 do array vai estar o ret do Carlos.

Quando o .map((el) => { ... }) roda, a variável el representa a caixa inteira que está dentro daquela gaveta.

O que tem dentro da caixa el?
Dentro dessa caixa (el), não importa em qual gaveta do array ela esteja, ela sempre vai ter a mesma estrutura interna que foi definida lá no molde da classe:

JavaScript
el = {
    nome: "Lucas",  // <-- O this.nome está aqui dentro!
    idade: "18",   // <-- O this.idade está aqui dentro!
    getnome: function() { ... },
    getidade: function() { ... }
}
Então, quando você digita el.getnome(), o JavaScript não está olhando se o nome é o "primeiro valor" do array. Ele simplesmente abre a caixa el, olha lá dentro e procura pela função getnome(), que por sua vez sabe exatamente como pegar a propriedade nome dentro daquela mesma caixa.
