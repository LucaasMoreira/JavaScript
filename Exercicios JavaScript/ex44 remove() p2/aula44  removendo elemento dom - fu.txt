===================================================================
MEUS APRENDIZADOS - JAVASCRIPT (MANIPULAÇÃO DE DOM E FILTROS)
===================================================================

1. RETORNO DE FUNÇÕES (Mecânica dos dois Returns):
   - O primeiro "return" (dentro do .filter) é booleano (true/false). Ele serve apenas para o motor do filter saber se guarda ou joga fora o elemento.
   - O segundo "return" (no final da função) é o que ejeta o dado para fora. É ele que entrega o elemento HTML destravado (ou undefined) para a variável do botão.
   - Diferença de performance: Usar "return radioSelecionado[0]" direto corta caminhos na memória do computador, economizando linhas de atribuição.

2. FLUXO DE EXECUÇÃO (Como o botão sabe quem foi clicado):
   - O botão sozinho é "cego". Ele não guarda o valor do rádio em tempo real.
   - A função "radioSelecionado" só acorda e roda no exato milissegundo em que o botão é clicado.
   - Ao ser chamada pelo clique do botão, a função faz uma varredura instantânea na tela para ver qual rádio está ativo.

3. TRATAMENTO DE ERROS E SEGURANÇA COM IF/ELSE:
   - Se o botão for clicado sem nenhum curso selecionado, o filter gera um array vazio [], e tentar ler a posição [0] resulta em "undefined".
   - Tentar ler propriedades (como .parentNode) de algo que é "undefined" quebra o código e gera o erro vermelho no console do navegador.
   - O "if (rs != undefined)" funciona como um desvio de segurança. Se for undefined, o JavaScript ignora a linha que daria erro e pula direto para o "else", limpando o erro do console.

4. LOGICA DE NAVEGAÇÃO: TARGET vs FILTER:
   - O "evt.target" serve para descobrir quem recebeu o clique direto naquele instante (ex: o próprio botão). Ele não serve para ler outros elementos longe dele.
   - O "filter + .checked" serve para fazer uma varredura geral na página e descobrir quem está ativo no HTML, não importando onde o usuário clicou.

5. COMPORTAMENTO DO RÁDIO BUTTON NO FILTER:
   - Se por um erro de lógica houvesse dois rádios marcados, o código pegaria apenas o primeiro, porque o "return [0]" está travado para resgatar sempre a primeira posição.
   - No rádio button isso não quebra o sistema porque a propriedade HTML "name" garante que apenas um rádio fique marcado por vez.
